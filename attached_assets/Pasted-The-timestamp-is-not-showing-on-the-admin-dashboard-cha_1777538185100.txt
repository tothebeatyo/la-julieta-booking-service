The timestamp is not showing on the admin dashboard 
chat history. Do the following to find and fix it:

STEP 1 — Find the exact file:
Search ALL files in artifacts/chatbot-landing/src/ 
for where messages are rendered. Look for any file 
containing:
- message.content
- message.text  
- messages.map
- direction === "outbound"
- "inbound"

List ALL files that match so we know exactly 
which component renders the chat bubbles.

STEP 2 — Check the API response:
In artifacts/api-server/src/routes/admin.ts,
find the GET /clients/:psid/messages route.

Make sure the SQL query explicitly includes created_at:
SELECT id, psid, direction, content, created_at 
FROM messages 
WHERE psid = $1 
ORDER BY created_at ASC

STEP 3 — Fix the message bubble component:
Once you find the file from Step 1, add this 
timestamp helper at the top of the file:

const formatTimestamp = (ts: string | null | undefined): string => {
  if (!ts) return "Unknown time";
  try {
    return new Date(ts).toLocaleString("en-PH", {
      timeZone: "Asia/Manila",
      month: "short",
      day: "numeric",
      year: "numeric",
      hour: "numeric",
      minute: "2-digit",
      hour12: true,
    });
  } catch {
    return "Unknown time";
  }
};

Then find where each message bubble is rendered 
and wrap it like this:

BEFORE (what it probably looks like):
<div className="message-bubble">
  {message.content}
</div>

AFTER (add timestamp below):
<div className="flex flex-col gap-1">
  <div className="message-bubble">
    {message.content}
  </div>
  <span 
    className={`text-xs text-gray-400 px-1 ${
      message.direction === "outbound" 
        ? "text-right self-end" 
        : "text-left self-start"
    }`}
  >
    {formatTimestamp(message.created_at)}
  </span>
</div>

STEP 4 — Add date divider between days:
Before rendering each message, check if the date 
changed from the previous message:

{messages.map((msg, index) => {
  const currentDate = msg.created_at 
    ? new Date(msg.created_at).toLocaleDateString("en-PH", { 
        timeZone: "Asia/Manila",
        month: "long", 
        day: "numeric", 
        year: "numeric" 
      })
    : null;
  
  const prevDate = index > 0 && messages[index-1].created_at
    ? new Date(messages[index-1].created_at).toLocaleDateString("en-PH", { 
        timeZone: "Asia/Manila",
        month: "long", 
        day: "numeric", 
        year: "numeric" 
      })
    : null;

  const showDateDivider = currentDate && currentDate !== prevDate;

  return (
    <div key={msg.id}>
      {showDateDivider && (
        <div className="flex items-center gap-2 my-4">
          <div className="flex-1 h-px bg-gray-200" />
          <span className="text-xs text-gray-500 font-medium px-2">
            {currentDate}
          </span>
          <div className="flex-1 h-px bg-gray-200" />
        </div>
      )}
      {/* message bubble with timestamp */}
    </div>
  );
})}

STEP 5 — Console log to verify data is coming through:
Add this temporarily to confirm created_at exists 
in the data:
console.log("Messages data:", messages);

STEP 6 — Rebuild frontend:
After all changes run:
cd artifacts/chatbot-landing && pnpm run build

Then restart the server.
